# App Store / Play Store "What's New" — release notes

Customer app release covering: new languages (Russian, Italian, German), faster
delivery (auto-dispatch), invite-a-friend referrals, and smoother hotel delivery.

**How to use:** paste the matching locale's text into App Store Connect →
*Version Information → What's New in This Version* (per language), and Play
Console → *Release → Release notes* (per language, inside `<xx-YY>` tags).

- Keep the **first 2–3 lines** strongest — that's all most users see before "…more".
- These are benefit-focused and avoid internal jargon (no "auto-dispatch",
  "RPC", etc.). Amounts (EGP 50) match the live `platform_settings` defaults.
- The store listing **subtitle/description** is separate — this is only the
  per-version "What's New" text.

## Notes for the submitter

- **Auto-dispatch ships dormant.** The "faster delivery" line is only true once
  you flip `platform_settings.dispatch_mode` to `auto` (or pilot a zone). Don't
  publish that line until the migration is deployed and the switch is on —
  otherwise it's a promise the build doesn't yet keep. If you submit the build
  before enabling auto-dispatch, drop the "Faster delivery" bullet from this
  version's notes and add it back when you turn it on.
- **Referrals + new languages are live as soon as the build ships** (no server
  switch needed beyond deploying migration 026 for referrals).
- Character budget: App Store "What's New" allows 4000 chars; Play Store allows
  500 per language. The short variant fits Play's limit comfortably.
